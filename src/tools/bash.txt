Execute bash commands in a persistent shell.

Usage:
- Use for terminal operations (git, npm, docker, gh).
- **DO NOT** use for file operations (read, write, edit, grep, glob) - use specialized tools instead.
- Use `workdir` parameter instead of `cd`.
- Quote paths with spaces: `"path with spaces/file"`.
- Timeouts: Defaults to 2 mins (120000ms).
- Parallelism: Run independent commands (e.g., `git status` + `git diff`) in parallel tool calls.
- **Truncation:** Large outputs are truncated in the response.

Git/PR Safety:
- No interactive flags (e.g., `-i`).
- No destructive commands (hard reset, force push) unless requested.
- No `git commit --amend` if already pushed or created by someone else.
- Use `gh` for GitHub tasks.
